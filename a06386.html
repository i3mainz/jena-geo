Detailed Description

Basic access to grid data values backed by a two-dimensional RenderedImage.

While images are two-dimensional, the coverage envelope may have more dimensions. In other words the rendered image can be a two-dimensional slice in a n-dimensional space. The only restriction is that the GridGeometry::getExtent() grid extent} has a GridExtent::getSize(int) size} equals to 1 in all dimensions except two of them.

Example: a remote sensing image may be valid only over some time range (the temporal period of the satellite passing over observed area). Envelopes for such grid coverage can have three dimensions: the two usual ones (horizontal extent along x and y), and a third dimension for start time and end time (temporal extent along t). This "two-dimensional" grid coverage can have any number of columns along x axis and any number of rows along y axis, but only one plan along t axis. This single plan can have a lower bound (the start time) and an upper bound (the end time).

Image size and location

The RenderedImage::getWidth() image width} and RenderedImage::getHeight() height} must be equal to the GridExtent::getSize(int) grid extent size} in the two dimensions of the slice. However the image origin ( RenderedImage::getMinX() minimal x} and RenderedImage::getMinY() y} values) does not need to be equal to the GridExtent::getLow(int) grid extent low values}; a translation will be applied as needed.

Image bands

Each band in an image is represented as a SampleDimension.

Constructs a grid coverage using the specified envelope, range and data.

This convenience constructor computes a GridGeometry from the given envelope and image size. This constructor assumes that all grid axes are in the same order than CRS axes and no axis is flipped. This straightforward approach often results in the y axis to be oriented toward up, not down as commonly expected with rendered images.

This constructor is generally not recommended because of the assumptions on axis order and directions. For better control, use the constructor expecting a GridGeometry argument instead. This constructor is provided mostly as a convenience for testing purposes.

Parameters
domainthe envelope encompassing all images, from upper-left corner to lower-right corner. If
null

rangesample dimensions for each image band. The size of this list must be equal to the number of bands. If
null
